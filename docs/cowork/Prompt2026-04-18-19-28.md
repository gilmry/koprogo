## Mission

Valider en 4 couches, dans l'ordre. Arrête-toi au premier échec bloquant et reporte.

### P0 — Smoke cross-cutting (5 min)

- [ ] `docker compose up -d` → les 5 services restent `Up` et `healthy`
- [ ] `http://localhost` charge la home en < 2s, aucune erreur console
- [ ] Switch FR → NL → DE → EN via le sélecteur de la Nav : la sidebar, les dashboards et les badges (tickets, résolutions, polls) suivent sans clé manquante
- [ ] Login `admin@koprogo.com / admin123` → dashboard SuperAdmin visible
- [ ] Déconnexion manuelle → **1 seul toast** "Session expirée" (pas de cascade — filet posé dans #383)

### P1 — Contrat de types (nouveau — Sprint 7)

Toute dérive entre enum Rust et enum Svelte doit désormais être impossible. Vérifier sur 3 parcours :

- [ ] **Tickets** : ouvrir modal de création → priorités `Low/Medium/High/Critical` (pas d'`Urgent` inventé), 9 catégories dont `CommonAreas/Elevator/Landscaping/Other`, statuts affichés = `Open/InProgress/Resolved/Closed/Cancelled` (**sans `Assigned`** — ce variant n'existe pas côté backend, dérivé de `assigned_to IS NOT NULL`)
- [ ] **Résolutions** : créer résolution → `ResolutionType` en snake_case (`ordinary/extraordinary`), `MajorityType` cohérent `simple/absolute/qualified`
- [ ] **Expenses** : liste → badges `approval_status` = `draft/pending_approval/approved/rejected` (backend truth)
- [ ] Sondage aléatoire : ouvrir n'importe quelle page enum-heavy, inspecter les dropdowns et les badges — **si un label n'apparaît pas, c'est une clé i18n manquante, pas un mismatch backend**

### P2 — Runes + WCAG 2.1 AA (nouveau — Sprint 6+8)

- [ ] Ouvrir `UnitCreateModal`, `OwnerCreateModal`, `MeetingCreateModal` successivement : le champ requis reçoit le focus initial, **Tab boucle dans le modal** (ne sort pas vers la page), **Escape ferme** ET le focus revient sur le bouton déclencheur
- [ ] `NotificationDropdown` (cloche en haut-droite) : clavier uniquement, `Arrow Up/Down` navigue entre items, `Home/End` saute aux extrêmes, `Escape` ferme
- [ ] Boutons icône (edit ✏️, delete 🗑️ dans `UnitOwners`, `QuoteList`, `TicketList`) : chaque bouton doit avoir un `aria-label` lisible à l'inspecteur DOM
- [ ] Formulaires : chaque `<select>` et `<input>` a un `<label for=id>` visible ou `sr-only` (tab clavier → le lecteur d'écran annonce le champ)
- [ ] Backdrop de modal : clic sur l'arrière-plan sombre ferme, Tab/Enter sur un élément focusable fonctionne aussi (fix #408)

### P3 — Parcours métier multi-rôles (régression v1→v7)

Les bugs v1→v7 étaient majoritairement dans ces parcours. Refais-les pour confirmer zéro régression.

- [ ] **Ticket full** : `charlie@owner.be` crée un ticket → syndic ouvre `TicketAssignModal` → **dropdown de contractors** (pas input UUID, fix #392) → sélectionne `marc@plomberie.be` (nouveau seed Sprint 7) → ticket passe Assigned → Start → Resolve → Close
- [ ] **AG + quorum + vote** : syndic crée AG → `QuorumPanel` visible, saisit 800/1000 → passe "Quorum validé" → crée résolution → owner vote Pour → syndic clôture → statut Adoptée. **Vérifier qu'aucun bouton syndic ne s'affiche quand l'owner visite la page** (fix B18, canManage gating)
- [ ] **Facture** : syndic crée facture → `BuildingSelector` visible (fix B3) → soumission OK → badge "En attente" traduit dans la liste
- [ ] **Erreur forcée** : provoquer un 500 (ex. mauvais ID) → toast "Une erreur est survenue", **jamais** un message Postgres brut (fix #383)

### P4 — Filet CI anti-drift (dry-run, sans PR)

Localement :

```bash
# Gate 1+2 : Rust → OpenAPI → TypeScript
make openapi-export && cd frontend && npm run types:generate
git diff --exit-code docs/api/openapi.json frontend/src/types/api.d.ts
# → doit être clean après pull

# Gate 3 : TS → Svelte
docker compose exec frontend sh -c "cd /app && npx svelte-check --threshold warning"
# → "0 errors and 0 warnings"

# Gate 4 : cap enums hand-written
grep -c "^export enum" frontend/src/lib/api/*.ts | awk -F: '{sum+=$2} END {print sum}'
# → ≤ 4 (actuellement 2 : AvailabilityStatus et LoanStatus dans sharing.ts, documentés)
```

Si un de ces gates échoue, c'est que la branche a dérivé depuis le push — signaler.
