### Fix #v3-1 — Suppression de la priorité "Urgent" (mismatch frontend/backend)

**Cause racine** : le backend Rust `TicketPriority` n'a que 4 variantes (`Low`, `Medium`, `High`, `Critical`). Le commentaire dans le code confirme : `Critical, // Critique/Urgente` → la notion "Urgente" est volontairement fusionnée avec Critical. Le frontend exposait 5 niveaux, dont un `Urgent` orphelin qui causait `400 unknown variant Urgent`.

**Fix** : suppression de `TicketPriority.Urgent` dans :
- [`frontend/src/lib/api/tickets.ts`](frontend/src/lib/api/tickets.ts) — enum
- [`frontend/src/components/tickets/TicketCreateModal.svelte`](frontend/src/components/tickets/TicketCreateModal.svelte) — option du dropdown
- [`frontend/src/components/tickets/TicketPriorityBadge.svelte`](frontend/src/components/tickets/TicketPriorityBadge.svelte) — entrée du badge map
- [`frontend/src/components/tickets/TicketList.svelte`](frontend/src/components/tickets/TicketList.svelte) — option du filtre

Les 4 niveaux restants (Basse 7j, Moyenne 3j, Haute 24h, Critique 1h) couvrent l'ensemble des besoins métier.

**À tester** :
1. Ouvrir la modale de création de ticket
