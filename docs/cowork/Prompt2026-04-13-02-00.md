## Points de vérification critiques

### Validation des fixes v6

**P2 Résolutions + Votes (B1 + B2)**

1. Syndic crée une AG → clic "+ Ajouter résolution" dans la page meeting-detail
2. **Vérifier** :
   - Dropdown "Type" a **2 options** : Ordinaire / Extraordinaire (plus de Standard/Budget/Travaux/etc.)
   - Dropdown "Majorité requise" a **4 options** : Absolue / 2/3 / 4/5 / Unanimité
3. Soumettre → POST `/meetings/{id}/resolutions` → **201 Created**
4. Connecte-toi en owner (Alice, Bob ou Charlie) → va sur la meeting-detail → voir la résolution
5. **Vérifier** : si l'owner a plusieurs lots, un sélecteur "Voter au nom du lot" apparaît
6. Cliquer Pour/Contre/Abstention, définir les tantièmes, soumettre → POST `/resolutions/{id}/vote` → **201 Created**
7. Vérifier le décompte affiché (Pour X votes Y%, Contre, Abstention, Total tantièmes)
8. Syndic retourne sur la page → clôturer le vote → statut passe à Adoptée/Rejetée
9. **Note** : il se peut que le backend refuse la création de résolution si le quorum n'a pas été validé (message "Quorum has not been validated yet"). Dans ce cas, valider le quorum au backend par un POST curl AVANT de tester (le fix UI est déféré) :
   ```bash
   curl -X POST http://localhost/api/v1/meetings/{MEETING_ID}/validate-quorum \
     -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
     -d '{"present_quotas": 800, "total_quotas": 1000}'
   ```
   **Et signaler dans le rapport** : "UI quorum toujours manquante (B4 déféré)".
