## Per-bug verdict

### BUG-WF14-2 — cross-org / cross-owner building leak (CRITICAL)

**Verdict: ALREADY-FIXED. Isolation holds. No leak reproducible.**

Traced the full path:

- Handler `building_handlers.rs::list_buildings` (l.124-155): extracts
  `organization_id` from the JWT-backed `AuthenticatedUser` (never from
  client input). For `role == "owner"` it passes `owner_user_id =
  Some(user.user_id)` into `list_buildings_paginated_for_user`.
- Use case `building_use_cases.rs::list_buildings_paginated_for_user`
  (l.72-91): builds `BuildingFilters { organization_id, owner_user_id }`.
- Repo `building_repository_impl.rs::find_all_paginated` (l.166-175):
  when `owner_user_id` is set, adds
  `id IN (SELECT DISTINCT u.building_id FROM units u
   JOIN unit_owners uo ON uo.unit_id = u.id
   JOIN owners o ON o.id = uo.owner_id
   WHERE o.user_id = $N AND uo.end_date IS NULL)`.
  Both COUNT and data queries bind the same param — pagination total is
  consistent. Sort column is whitelisted (no SQL injection vector).
- `AuthenticatedUser` (middleware.rs l.32-72): `organization_id` and `role`
  come from verified JWT claims; `verify_org_access` / `require_organization`
  enforce tenant scoping on the by-id path too.

Schema confirmed: `owners.user_id` (migration 20251031114200) and
`unit_owners.end_date` nullable (migration 20250127000000) both exist, so
the active-ownership filter is sound.

Fix landed in commit `dddde26` ("fix(backend): … building isolation by
role"), 2026-04-01 — the SAME day as the review, so the report never saw it.
Confirmed `git merge-base --is-ancestor dddde26 HEAD` → in `feature/dev`.

**Gap found & closed:** the existing `multitenancy.feature` only covered
cross-*organization* filtering. The report's exact case (owner sees OTHER
buildings of the SAME org) had no regression guard. Added a RED-first
`@security` scenario "Owner sees only buildings where they own a unit
(BUG-WF14-2)" + 5 step impls in `bdd.rs`, exercising the production
`list_buildings_paginated_for_user(.., Some(owner_user_id))` path: org with
3 buildings, Alice owns a unit in only 1 → asserts she sees exactly 1.
